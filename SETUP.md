# SwipeToPopApp セットアップガイド

## 必要な環境
- macOS 12.0以降
- Xcode 14.0以降
- iOS 15.0以降対応デバイスまたはシミュレータ

## セットアップ手順

### 1. プロジェクトのクローン
```bash
git clone https://github.com/yomi4486/swipe_to_pop.git
cd swipe_to_pop
```

### 2. Xcodeでプロジェクトを開く
```bash
open SwipeToPopApp.xcodeproj
```

### 3. プロジェクト設定
1. Xcodeでプロジェクトファイルを選択
2. ターゲット「SwipeToPopApp」を選択
3. 「Signing & Capabilities」タブで開発チームを設定
4. Bundle Identifierを一意の値に変更（例: com.yourname.SwipeToPopApp）

### 4. ビルドと実行
1. シミュレータまたは実機を選択
2. ⌘+R でビルド・実行

## 動作確認

### 写真ライブラリアクセス許可
初回起動時に写真ライブラリへのアクセス許可を求められます。「許可」を選択してください。

### デモモード
写真ライブラリアクセスなしでアプリを試したい場合は、メイン画面で「デモを試す」を選択してください。

## トラブルシューティング

### ビルドエラーが発生する場合
1. Xcodeのバージョンが14.0以降であることを確認
2. iOS Deployment Targetが15.0に設定されていることを確認
3. Swift Versionが5.0以降に設定されていることを確認

### 写真が表示されない場合
1. 写真ライブラリへのアクセス許可が与えられていることを確認
2. デバイス/シミュレータに写真が保存されていることを確認

### パフォーマンスの問題
1. 大量の写真（10,000枚以上）がある場合、初回読み込みに時間がかかることがあります
2. 実機での動作を推奨します（シミュレータは動作が重い場合があります）

## 機能説明

### メイン画面
- 中央に現在の画像が表示
- 左右にスワイプアクションの説明
- 右上に設定ボタン
- 上部に進捗表示

### スワイプ操作
- **左スワイプ**: デフォルトで削除（設定で変更可能）
- **右スワイプ**: デフォルトでお気に入り（設定で変更可能）
- **閾値**: 100ポイント以上のスワイプでアクション実行

### 設定画面
- スワイプアクションのカスタマイズ
- フォルダ管理
- 統計情報表示
- アプリ情報

### アクション詳細
1. **削除**: 写真を完全に削除（復元不可）
2. **非表示**: アプリ内でのみ非表示（元の写真は保持）
3. **フォルダ移動**: 選択したアルバムに写真を追加
4. **お気に入り**: iOS標準のお気に入り機能を使用
