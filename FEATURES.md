# SwipeToPopApp 機能仕様書

## アプリ概要
マッチングアプリのような直感的なスワイプ操作で写真を効率的に整理できるiOSアプリ。

## 主要機能

### 1. 画像表示・ナビゲーション
- 写真ライブラリから画像を読み込み
- 作成日の新しい順で表示
- カード型UI（Shadow、角丸）
- 高品質画像表示（800x800ピクセル）
- プログレスインジケーター表示

### 2. スワイプジェスチャー
- ドラッグによるカード移動
- 回転エフェクト（スワイプ量に応じて）
- スプリングアニメーション
- 閾値ベースアクション実行（100ポイント）
- リアルタイムビジュアルフィードバック

### 3. カスタマイズ可能なアクション
#### 左スワイプ（デフォルト：削除）
- 削除：PHAssetChangeRequest使用
- 非表示：アプリ内フラグ管理
- フォルダ移動：カスタムアルバム作成・移動
- お気に入り：iOS標準機能連携

#### 右スワイプ（デフォルト：お気に入り）
- 同様の4種類のアクションから選択可能

### 4. フォルダ管理機能
- カスタムフォルダ作成・削除
- デフォルトフォルダ（Sorted, Favorites, Archive, Work, Personal）
- UserDefaultsによる設定保存
- PHAssetCollectionとの連携

### 5. 設定画面
- セグメント型ピッカーによるアクション選択
- フォルダ管理UI
- 統計情報表示
- 操作説明・ヘルプ

### 6. デモモード
- 写真ライブラリアクセス不要
- グラデーション付きカード表示
- 全機能のシミュレーション
- 設定変更対応

## 技術仕様

### アーキテクチャ
- **フレームワーク**: SwiftUI
- **パターン**: MVVM
- **データバインディング**: @ObservableObject, @Published
- **状態管理**: Combine

### 使用フレームワーク
- **SwiftUI**: UI構築
- **Photos**: 写真ライブラリアクセス
- **Foundation**: データ管理
- **UserDefaults**: 設定保存

### パフォーマンス最適化
- 遅延ローディング（onAppear）
- 適切な画像サイズ（800x800）
- メモリ効率的な画像管理
- バックグラウンド処理（DispatchQueue.main.async）

### パーミッション
- **NSPhotoLibraryUsageDescription**: 画像表示・整理
- **NSPhotoLibraryAddUsageDescription**: お気に入り・アルバム追加

## ユーザビリティ

### 直感的操作
- マッチングアプリライクなスワイプ
- リアルタイムプレビュー
- 明確なビジュアルフィードバック
- 一貫したアニメーション

### アクセシビリティ
- セマンティックな色使い（赤=削除、緑=お気に入り）
- アイコンとテキストの併用
- 適切なフォントサイズ
- コントラスト配慮

### エラーハンドリング
- パーミッション拒否時の適切な案内
- 写真なし状態の処理
- ネットワーク画像の読み込み失敗対応
- グレースフルな操作失敗処理

## データフロー

### 画像読み込み
1. PHPhotoLibrary.requestAuthorization
2. PHAsset.fetchAssets (sortDescriptors)
3. ImageItem配列生成
4. PHImageManager.requestImage

### スワイプアクション
1. DragGesture検知
2. アクション判定（左右・閾値）
3. performSwipeAction実行
4. 対象アクション処理
5. moveToNextImage

### 設定保存
1. UserDefaults書き込み
2. @Published更新
3. UI自動更新
4. 設定の永続化

## セキュリティ・プライバシー
- 最小限のパーミッション要求
- ローカル処理のみ（ネットワーク送信なし）
- ユーザー制御による削除確認
- 適切なInfo.plist記述
